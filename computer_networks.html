The 17 questions a CN round actually opens with. Lead with the one-line definition, give one example, then handle the follow-up before they ask.
01Walk me through the OSI model.Must know
15-sec
7 layers, each one job: Application (HTTP, DNS), Presentation (encryption), Session (connections), Transport (TCP/UDP, ports), Network (IP, routing), Data Link (MAC, frames), Physical (bits). Mnemonic: All People Seem To Need Data Processing. Full
OSI is a 7-layer reference model for how data moves across a network, each layer with one job. From the top: Application holds user protocols like HTTP/DNS; Presentation handles encryption and formatting; Session manages connections; Transport gives end-to-end delivery with TCP/UDP and ports; Network handles logical addressing and routing with IP; Data Link frames data using MAC addresses; Physical is raw bits on the wire. Follow-up
Which layer does a router work at, and a switch? Router = L3 (Network, IP); switch = L2 (Data Link, MAC).
02OSI vs TCP/IP model?Must know
15-sec
OSI is the 7-layer theory model; TCP/IP is the 4-layer model the internet actually runs on. TCP/IP merges OSI's top three into one Application layer and the bottom two into Network Access. Full
Both are layered, but OSI is the 7-layer theoretical reference while TCP/IP is the 4-layer implementation. TCP/IP collapses Application + Presentation + Session into one Application layer, and Physical + Data Link into Network Access. So TCP/IP = Application, Transport, Internet, Network Access. OSI teaches; TCP/IP runs. Follow-up
Why did TCP/IP win? It shipped with working protocols and the actual internet; OSI stayed a reference.
03TCP vs UDP — when do you use which?Must know
15-sec
TCP is connection-oriented and reliable (handshake, ordering, retransmission) — web, email, files. UDP is connectionless and best-effort but fast — video calls, streaming, games, DNS. Full
Both are Transport-layer with opposite trade-offs. TCP sets up a connection, guarantees ordered error-checked delivery, and does flow + congestion control — slower but dependable. UDP has no handshake, ordering, or retransmission — just fast lightweight datagrams. Use TCP when every byte must arrive; UDP when speed beats completeness. Follow-up
Why is the TCP header bigger? TCP needs ≥20 bytes for sequence numbers, acks, and window size; UDP needs only 8 because it tracks none of that.
04Explain the TCP 3-way handshake.Must know
15-sec
Client sends SYN, server replies SYN-ACK, client sends final ACK. Three messages so both sides agree on starting sequence numbers and the connection opens. Full
It opens a reliable connection and syncs sequence numbers. Client sends SYN with its initial sequence number. Server replies SYN-ACK: acknowledges the client's SYN and sends its own. Client sends a final ACK. Now both sides agree on starting sequence numbers and data flows. Follow-up
Why not two-way? Both sides must confirm the other can send and receive; two messages prove only one direction.
05Why does TCP teardown take 4 steps?Good to know
15-sec
Closing is per-direction because TCP is full-duplex. Each side sends its own FIN and gets its own ACK — four messages. Setup can combine the server's SYN and ACK into one. Full
A TCP connection is full-duplex, so each side closes its half separately. One side sends FIN, the other ACKs it, but that side may still have data, so its own FIN comes later as a separate message, which the first side ACKs — four messages total. Follow-up
What is TIME_WAIT? After the final ACK the closer waits ~2× the max segment lifetime so delayed packets die out before the port is reused.
06What happens when you type google.com and press Enter?Must know
15-sec
DNS resolves the name to an IP, TCP connects on port 443 (3-way handshake), TLS sets up encryption, the browser sends HTTP GET, the server returns HTML, the browser renders and fetches CSS/JS/images. Full
The browser resolves the name (cache → resolver → root → .com TLD → Google's authoritative server). It opens a TCP connection on port 443 with the 3-way handshake, does a TLS handshake, sends an HTTP GET, receives the HTML, then parses and fetches CSS/JS/images and renders. ARP and routing move packets hop by hop underneath. Follow-up
Where does ARP fit? At each local hop, ARP resolves the next hop's IP to its MAC so the frame can be addressed.
07How does DNS resolution work?Must know
15-sec
Machine checks its cache, then asks a recursive resolver, which walks root → .com TLD → authoritative server to get the IP, caches it for the TTL, and returns it. Full
DNS turns a name into an IP. Your machine checks its local cache, then asks a recursive resolver (usually your ISP's). If uncached, the resolver queries the hierarchy: root points to the .com TLD server, which points to the authoritative name server, which returns the IP. The resolver caches it for its TTL. Follow-up
TCP or UDP? Mostly UDP on port 53; falls back to TCP for large responses or zone transfers.
08HTTP vs HTTPS, and what does TLS add?Must know
15-sec
HTTP sends plaintext; HTTPS is HTTP over TLS, adding encryption, authentication (server certificate), and integrity. HTTPS = port 443, HTTP = port 80. Full
HTTP requests/receives web pages in plaintext, so anyone in between can read or tamper. HTTPS runs HTTP over TLS, adding encryption (unreadable in transit), authentication (you're talking to the real server via its certificate), and integrity (tampering detected). Follow-up
Symmetric or asymmetric? Both: the handshake uses asymmetric keys to agree on a shared secret, then the data uses fast symmetric encryption.
09Flow control vs congestion control?Good to know
15-sec
Flow control stops a fast sender from overwhelming a slow receiver (advertised window). Congestion control stops senders from overwhelming the network (congestion window). Full
Both throttle the sender but protect different things. Flow control uses the receiver's advertised window (free buffer space). Congestion control uses a congestion window managed by slow start and AIMD (additive-increase, multiplicative-decrease). Follow-up
What does the sender actually send? The minimum of the receiver window and the congestion window.
10Hub vs switch vs router?Must know
15-sec
Hub = L1, floods every port. Switch = L2, forwards a frame by MAC to the right port. Router = L3, forwards packets between networks by IP. Full
A hub (L1) repeats bits out every port — floods and shares bandwidth, basically obsolete. A switch (L2) learns MAC addresses and forwards a frame only to the correct port. A router (L3) connects different networks and forwards packets between them using IP and a routing table. Follow-up
Which gives separate collision domains? A switch — each port is its own; a hub puts everyone in one.
11IP addressing — public vs private, IPv4 vs IPv6.Must know
15-sec
IPv4 = 32 bits, ~4.3 billion (ran out). Private ranges (10/8, 172.16/12, 192.168/16) are reused locally; public addresses are globally unique. IPv6 = 128 bits, ~unlimited. Full
An IP identifies a device. IPv4 is 32 bits (e.g. 192.168.1.1), ~4.3 billion addresses, which ran out. Private ranges are reused inside local networks and never routed publicly. IPv6 is 128 bits in hex, with practically unlimited space — the long-term fix. Follow-up
How do private addresses reach the internet? Through NAT at the router.
12What is subnetting and CIDR?Must know
15-sec
Subnetting splits a network by borrowing host bits for the network part. CIDR writes the split as a slash: /24 = 256 addresses, 254 usable after network + broadcast. Full
Subnetting splits one large network into smaller ones by borrowing bits from the host part. CIDR like /24 means the first 24 bits are the network. A /24 gives 256 total, 254 usable (one network, one broadcast). It improves organisation, security, and routing. Follow-up
Usable hosts in a /26? 2^(32-26) - 2 = 62.
13What is ARP and why is it needed?Good to know
15-sec
ARP maps a Layer-3 IP to a Layer-2 MAC on the local network. A device broadcasts "who has this IP", the owner replies with its MAC, and the sender caches it. Full
Once a packet reaches the local network, frame delivery uses MAC, not IP. So a device broadcasts an ARP request for an IP, the owner replies with its MAC, and the sender caches the mapping in its ARP table. Follow-up
Difference from DNS? DNS maps name→IP; ARP maps IP→MAC. Different layers, different jobs.
14What does DHCP do?Good to know
15-sec
Auto-assigns IP config when a device joins, via DORA: Discover, Offer, Request, Acknowledge. The lease includes IP, mask, gateway, and DNS, and renews over time. Full
DHCP assigns IP configuration automatically. Four steps (DORA): client broadcasts Discover, servers send an Offer, client sends a Request for one offer, server sends an Acknowledge to lease it. Ports 67 (server) / 68 (client). Follow-up
If the DHCP server is down? The device may fall back to a link-local APIPA address (169.254.x.x) and can't reach the wider network.
15What is NAT and why do we use it?Good to know
15-sec
NAT lets many private devices share one public IP. The router rewrites the private source address on outgoing packets and tracks the mapping by port, so replies return to the right device. Conserves scarce IPv4. Full
NAT lets a private network share one public IP. The router rewrites the private source address to its public one and tracks the mapping (usually by port), so replies reach the right internal device. It conserves IPv4 and hides the internal network. Follow-up
How does the router know which device a reply belongs to? A translation table keyed by the public port it assigned — that's PAT.
16Walk me through HTTP methods and status codes.Good to know
15-sec
Methods state intent: GET reads, POST creates, PUT replaces, PATCH partially updates, DELETE removes. Codes: 2xx success, 3xx redirect, 4xx client error, 5xx server error. Full
GET reads, POST creates, PUT replaces, PATCH partially updates, DELETE removes. Status ranges: 2xx success (200 OK), 3xx redirect (301), 4xx client error (404, 401), 5xx server error (500). First digit = category. Follow-up
PUT vs POST? PUT is idempotent (twice = same result); POST can create a new resource each time, so it isn't.
17Stop-and-Wait vs Go-Back-N vs Selective Repeat?Rare / deep
15-sec
Stop-and-Wait sends one frame and waits. Go-Back-N sends a window and resends the lost frame plus everything after it. Selective Repeat sends a window but resends only the lost frame, buffering the rest. Full
Sliding-window protocols differing in how much is in flight and how errors are handled. Stop-and-Wait: one frame at a time, simple but slow. Go-Back-N: window; on loss, retransmit that frame and all after it (receiver discards out-of-order). Selective Repeat: window; retransmit only the lost frame, buffering out-of-order ones — efficient but needs receiver memory. Follow-up
Which does TCP resemble? Selective Repeat, using selective acknowledgements.

Common traps that cost easy points Avoid

TrapReality
Calling TCP open a 4-way handshakeOpening is 3-way; closing is 4-way.
"UDP is reliable, it mostly arrives"UDP gives no delivery, ordering, or duplicate guarantees.
Forgetting to subtract 2 in host countA /26 has 64 addresses but only 62 usable.
Mixing flow and congestion controlReceiver vs network — keep them separate.
"DNS is always UDP"UDP for normal queries, TCP for zone transfers / large responses.
"HTTPS uses only asymmetric"Asymmetric sets up a symmetric key, which does the bulk work.
"IPv6 exists for speed"It exists because IPv4 addresses ran out.

Memorize cold

  • OSI = 7 layers (All People Seem To Need Data Processing).
  • TCP = reliable, ordered, 3-way open / 4-way close. UDP = fast, best-effort.
  • Switch = L2 (MAC), Router = L3 (IP).
  • Usable hosts = 2^(32 - prefix) - 2.
  • Flow control protects the receiver; congestion control protects the network.

Numbers to know

  • TCP header 20 bytes, UDP 8 bytes.
  • IPv4 32 bits, IPv6 128 bits.
  • Ports: HTTP 80, HTTPS 443, DNS 53, DHCP 67/68, SSH 22, SMTP 25.

One-liners to fire back

  • PDUs: data, segment, packet, frame, bits.
  • Handshake: SYN, SYN-ACK, ACK. Teardown: FIN, ACK, FIN, ACK.
  • Private: 10/8, 172.16/12, 192.168/16.
  • HTTPS = HTTP + TLS on 443.
