SLIDING WINDOW

Fixed and variable-size windows
CORE IDEA: Maintain a contiguous window [left, right]. Expand right to include, shrink left to satisfy a constraint. Turns O(n²) substring/subarray problems into O(n).
1
Fixed-Size Window
Time O(n)Space O(1)
Questions
1.Maximum Average Subarray I
2.Find All Anagrams in a String
3.Permutation in String
4.Sliding Window Maximum (deque)
Keyword: "subarray/substring of size k", "fixed length", "all windows of length k". Window size never changes — slide it.
Pseudocode
int windowSum = 0; for (int i=0; i<k; i++) windowSum += nums[i]; // first int best = windowSum; for (int i=k; i<n; i++) { windowSum += nums[i] - nums[i-k]; // add new, drop old best = Math.max(best, windowSum); } // Anagram: keep a 26-char count array, compare each slide
Pattern: Add incoming element, remove outgoing element (i-k). Constant-time window update.
2
Variable-Size Window (longest/shortest)
Time O(n)Space O(k)
Questions
1.Longest Substring Without Repeating Chars
2.Minimum Window Substring
3.Longest Repeating Character Replacement
4.Fruit Into Baskets
5.Minimum Size Subarray Sum
Keyword: "longest/shortest substring", "at most K distinct", "smallest subarray ≥ target". Window grows and shrinks dynamically.
Pseudocode
int left = 0, best = 0; Map<Character,Integer> count = new HashMap<>(); for (int right=0; right<n; right++) { count.merge(s.charAt(right), 1, Integer::sum); // expand while (/* window invalid */ count.size() > k) { char lc = s.charAt(left); // shrink count.merge(lc, -1, Integer::sum); if (count.get(lc)==0) count.remove(lc); left++; } best = Math.max(best, right - left + 1); // record }
Pattern: for-loop expands right; inner while shrinks left until valid. For "longest" → record after shrink; for "shortest" → record inside shrink loop.
3
Exact K / Count Subarrays
Time O(n)Space O(k)
Questions
1.Subarrays with K Different Integers
2.Binary Subarrays With Sum
3.Count Number of Nice Subarrays
4.Max Consecutive Ones III
5.Count Complete Subarrays in an Array
6.Number of Substrings Containing All Three Characters
Keyword: "exactly K", "count subarrays", "at most K". Convert exact counts to atMost(K) - atMost(K-1).
Pseudocode
int exactK(int[] nums, int k) { return atMost(nums, k) - atMost(nums, k-1); } int atMost(int[] nums, int k) { int left=0, ans=0; Map<Integer,Integer> freq = new HashMap<>(); for (int right=0; right<nums.length; right++) { if (freq.getOrDefault(nums[right],0)==0) k--; freq.put(nums[right], freq.getOrDefault(nums[right],0)+1); while (k < 0) { freq.put(nums[left], freq.get(nums[left])-1); if (freq.get(nums[left++])==0) k++; } ans += right - left + 1; } return ans; }
Pattern: Every valid window ending at right contributes right-left+1 subarrays. Exact K is usually two at-most passes.
4
Monotonic Deque Windows
Time O(n)Space O(k)
Questions
1.Sliding Window Maximum
2.Shortest Subarray with Sum at Least K
3.Constrained Subsequence Sum
4.Jump Game VI
5.Longest Continuous Subarray With Absolute Diff <= Limit
Keyword: "window max/min", "best previous within k", "absolute diff limit". Deque stores useful candidates in sorted order.
Pseudocode
// Sliding Window Maximum Deque<Integer> dq = new ArrayDeque<>(); // indices, values decreasing for (int i=0; i<n; i++) { while (!dq.isEmpty() && dq.peekFirst() <= i-k) dq.pollFirst(); while (!dq.isEmpty() && nums[dq.peekLast()] <= nums[i]) dq.pollLast(); dq.addLast(i); if (i >= k-1) ans.add(nums[dq.peekFirst()]); }
Pattern: Remove expired indices from the front; remove dominated candidates from the back. Each index enters and leaves once.
