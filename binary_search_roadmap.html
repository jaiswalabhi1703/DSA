BINARY SEARCH

Boundary, rotated arrays, search-on-answer
CORE IDEA: Halve the search space each step. Works on sorted arrays AND on "answer space" (binary search the answer). The hard part is defining the predicate that's monotonic (false…false, true…true).
1
Classic + Boundary Search
Time O(log n)Space O(1)
Questions
1.Binary Search (basic)
2.First & Last Position (lower/upper bound)
3.Search Insert Position
4.Find Peak Element
Keyword: "sorted array", "find target", "first/last occurrence", "O(log n) required". lower_bound = first index ≥ target.
Pseudocode
// Lower bound: first index where nums[i] >= target int lo=0, hi=n; // note hi=n (exclusive) while (lo < hi) { int mid = lo + (hi-lo)/2; // avoid overflow if (nums[mid] < target) lo = mid+1; else hi = mid; } return lo; // insertion point / first ≥ target // Upper bound: change condition to nums[mid] <= target
Pattern: Use half-open [lo, hi). lo + (hi-lo)/2 prevents overflow. Lower/upper bound = the building blocks for counting occurrences.
2
Rotated / Modified Array
Time O(log n)Space O(1)
Questions
1.Search in Rotated Sorted Array
2.Find Minimum in Rotated Sorted Array
3.Search in Rotated Array II (dupes)
4.Single Element in Sorted Array
Keyword: "rotated sorted array", "mountain array", "find min/pivot". One half is always sorted — decide which.
Pseudocode
int lo=0, hi=n-1; while (lo <= hi) { int mid = lo + (hi-lo)/2; if (nums[mid]==target) return mid; if (nums[lo] <= nums[mid]) { // left half sorted if (nums[lo]<=target && target<nums[mid]) hi=mid-1; else lo=mid+1; } else { // right half sorted if (nums[mid]<target && target<=nums[hi]) lo=mid+1; else hi=mid-1; } }
Pattern: Compare nums[lo] vs nums[mid] to find the sorted half, then check if target lies in that half's range.
3
Binary Search on Answer
Time O(n log M)Space O(1)
Questions
1.Koko Eating Bananas
2.Capacity to Ship Packages in D Days
3.Split Array Largest Sum
4.Median of Two Sorted Arrays
5.Minimize Max Distance to Gas Station
Keyword: "minimum/maximum X such that...", "minimize the largest", "can we do it in K?". Binary search the answer value, not an index. M = value range.
Pseudocode
// Find smallest feasible answer in [lo, hi] int lo = minPossible, hi = maxPossible; while (lo < hi) { int mid = lo + (hi-lo)/2; if (canDo(mid)) // monotonic predicate hi = mid; // mid works → try smaller else lo = mid + 1; // mid fails → need larger } return lo; // canDo(speed): simulate, return true if feasible boolean canDo(int x) { /* O(n) check */ }
Pattern: Define a monotonic canDo(x) predicate. Binary search the minimum x where it flips false→true. Each check is O(n) → total O(n log M).
