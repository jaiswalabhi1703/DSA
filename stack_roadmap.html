STACK PATTERNS ROADMAP (PHASE-WISE)

All stack patterns end-to-end · 8 phases · Java pseudocode
CORE IDEA

Stack = Last In First Out (LIFO). Use when you need to remember previous state while processing forward.
Key insight: Stack excels at problems where you need the nearest greater/smaller element,
tracking unmatched pairs, or evaluating nested/sequential expressions.

Stack Pattern Recognition
  • Monotonic Stack → next/prev greater or smaller element
  • Matching/Balancing → brackets, tags, pairs
  • Calculator → evaluate expressions with precedence
  • Deque/Sliding Window → max/min in window O(n)
  • Stack Design → min-stack, max-stack, special ops
  • Tree/Graph DFS → iterative traversal via stack
  • Histogram / Rectangle → largest area problems
When to reach for a Stack?
  • "Next greater / previous smaller element"
  • "Valid parentheses / balanced brackets"
  • "Evaluate expression / implement calculator"
  • "Largest rectangle / trapping rain water"
  • "Undo / backtrack last action"
  • "Iterative DFS on tree or graph"
  • "Sliding window min/max" → use Deque
Monotonic Increasing: bottom→top ascending (prev smaller)
Monotonic Decreasing: bottom→top descending (prev greater)

Monotonic Decreasing Stack: elements on stack go top→bottom in increasing order (largest at bottom).
When a larger element arrives → pop everything smaller → those popped elements found their NGE.
Next Greater Element template
// NGE: for each element, find next element greater than it int[] nge = new int[n]; Arrays.fill(nge, -1); Deque<Integer> stack = new ArrayDeque<>(); // stores INDICES for (int i=0; i<n; i++) { // pop all elements smaller than current while (!stack.isEmpty() && nums[stack.peek()] < nums[i]) { int idx = stack.pop(); nge[idx] = nums[i]; // nums[i] is NGE for idx } stack.push(i); } // remaining in stack have no NGE → stays -1 // Daily Temperatures variant: store index, ans = i - idx // CIRCULAR array (NGE II): iterate 2n, use i % n for (int i=0; i<2*n; i++) { while (!stack.isEmpty() && nums[stack.peek()] < nums[i%n]) nge[stack.pop()] = nums[i%n]; if (i < n) stack.push(i); }
Pattern: Maintain a monotonic decreasing stack (by value). When nums[i] breaks the monotonicity → pop and record answer. Always push index, not value.

Monotonic Increasing Stack: top→bottom descending (smallest at top).
When a smaller element arrives → pop all larger elements → those found their NSE.
Previous Smaller + Next Smaller template
// Previous Smaller Element (PSE) int[] pse = new int[n]; Deque<Integer> st = new ArrayDeque<>(); for (int i=0; i<n; i++) { while (!st.isEmpty() && nums[st.peek()] >= nums[i]) st.pop(); pse[i] = st.isEmpty() ? -1 : st.peek(); // peek not pop! st.push(i); } // Next Smaller Element (NSE) — process right to left int[] nse = new int[n]; st.clear(); for (int i=n-1; i>=0; i--) { while (!st.isEmpty() && nums[st.peek()] >= nums[i]) st.pop(); nse[i] = st.isEmpty() ? n : st.peek(); st.push(i); } // Remove K Digits (greedy with increasing stack): for (char c : num.toCharArray()) { while (k>0 && !st.isEmpty() && st.peek()>c) { st.pop(); k--; // remove larger digit } st.push(c); }
Pattern: Maintain a monotonic increasing stack (by value). For PSE: peek top after popping. For NSE: iterate right-to-left. Boundary sentinel: use -1 or n.

Longest Valid Parentheses trick: push -1 as base sentinel index. Answer = i - stack.peek() at each ')'.
Matching templates
// Valid Parentheses Deque<Character> st = new ArrayDeque<>(); for (char c : s.toCharArray()) { if (c=='('||c=='['||c=='{') st.push(c); else { if (st.isEmpty()) return false; char top = st.pop(); if (c==')' && top!='(') return false; if (c==']' && top!='[') return false; if (c=='}' && top!='{') return false; } } return st.isEmpty(); // Longest Valid Parentheses Deque<Integer> idxSt = new ArrayDeque<>(); idxSt.push(-1); int maxLen=0; // sentinel for (int i=0; i<n; i++) { if (s.charAt(i)=='(') { idxSt.push(i); } else { idxSt.pop(); if (idxSt.isEmpty()) idxSt.push(i); // new base else maxLen=Math.max(maxLen, i-idxSt.peek()); } } // Decode String k[encoded_string] Deque<Integer> counts = new ArrayDeque<>(); Deque<StringBuilder> builders = new ArrayDeque<>(); StringBuilder cur = new StringBuilder(); int k=0; for (char c : s.toCharArray()) { if (Character.isDigit(c)) k=k*10+(c-'0'); else if (c=='[') { counts.push(k); builders.push(cur); cur=new StringBuilder(); k=0; } else if (c==']') { int t=counts.pop(); StringBuilder prev=builders.pop(); for(int i=0;i<t;i++) prev.append(cur); cur=prev; } else cur.append(c); }
Pattern: Push opening brackets/indices. On closing bracket → pop and validate/compute. Stack empty at end = balanced. Sentinel index -1 enables length calculation.

Basic Calc II key insight: process operator of previous token, not current. Push positive/negative values for +/-. Push result of */÷ replacing top of stack.
Calculator II + RPN templates
// Basic Calculator II (+,-,*,/) — one stack Deque<Integer> st = new ArrayDeque<>(); int num=0; char sign='+'; for (int i=0; i<s.length(); i++) { char c = s.charAt(i); if (Character.isDigit(c)) num = num*10+(c-'0'); if ((!Character.isDigit(c) && c!=' ') || i==s.length()-1) { if (sign=='+') st.push(num); else if (sign=='-') st.push(-num); else if (sign=='*') st.push(st.pop()*num); else if (sign=='/') st.push(st.pop()/num); sign=c; num=0; } } int ans=0; while(!st.isEmpty()) ans+=st.pop(); // Evaluate Reverse Polish Notation (RPN) Deque<Integer> rpn = new ArrayDeque<>(); for (String t : tokens) { if ("+-*/".contains(t)) { int b=rpn.pop(), a=rpn.pop(); if (t.equals("+")) rpn.push(a+b); else if (t.equals("-")) rpn.push(a-b); else if (t.equals("*")) rpn.push(a*b); else rpn.push(a/b); } else rpn.push(Integer.parseInt(t)); } return rpn.pop();
Pattern: For +/−: push signed value. For */÷: pop top, compute, push result. Process operator of previous sign when new operator/end is reached. RPN: two pops per operator.

Maximal Rectangle: for each row build a histogram (heights[j] = 0 if cell==0 else heights[j]+1), then apply Largest Rectangle in Histogram.

Trapping Rain Water: for each bar, water = min(maxLeft, maxRight) - height[i]. Stack approach works in one pass.
Largest Rectangle + Rain Water templates
// Largest Rectangle in Histogram // Stack stores indices of increasing heights Deque<Integer> st = new ArrayDeque<>(); int[] h = Arrays.copyOf(heights, n+1); // append 0 sentinel int maxArea = 0; for (int i=0; i<=n; i++) { while (!st.isEmpty() && h[i] < h[st.peek()]) { int height = h[st.pop()]; int width = st.isEmpty() ? i : i-st.peek()-1; maxArea = Math.max(maxArea, height*width); } st.push(i); } // Trapping Rain Water — stack approach (one pass) Deque<Integer> tw = new ArrayDeque<>(); int water = 0; for (int i=0; i<n; i++) { while (!tw.isEmpty() && height[i] > height[tw.peek()]) { int bottom = tw.pop(); if (tw.isEmpty()) break; int bounded = Math.min(height[tw.peek()], height[i]) - height[bottom]; water += bounded * (i - tw.peek() - 1); } tw.push(i); }
Pattern: Histogram = NSE on both sides. Width = nse[i] - pse[i] - 1. Append 0-height sentinel to flush remaining stack at end. Rain water: track bounded height between walls.

Deque stores indices in decreasing order of value (for max).
Front = index of current window max.
Evict from front when out of window. Evict from back when new element ≥ back.
Sliding Window Maximum template
// Monotonic Deque — front always holds window max index int[] result = new int[n-k+1]; Deque<Integer> dq = new ArrayDeque<>(); // stores indices for (int i=0; i<n; i++) { // 1. Remove indices outside window while (!dq.isEmpty() && dq.peekFirst() < i-k+1) dq.pollFirst(); // 2. Remove from back elements smaller than current while (!dq.isEmpty() && nums[dq.peekLast()] < nums[i]) dq.pollLast(); dq.offerLast(i); // 3. Window fully formed → record max if (i >= k-1) result[i-k+1] = nums[dq.peekFirst()]; } // Sliding Window MINIMUM: flip comparison // while (!dq.isEmpty() && nums[dq.peekLast()] > nums[i]) // dq.pollLast(); → front holds min
Pattern: pollFirst() for out-of-window. pollLast() to maintain monotonicity. peekFirst() = current window max. Flip comparison sign for min variant.

Min Stack: store pair (value, currentMin) at each level.
Queue via 2 stacks: push to in, pour to out lazily on pop.
Amortized O(1) per op.
Min Stack + Queue via 2 Stacks
// Min Stack — store (val, minSoFar) pairs Deque<int[]> st = new ArrayDeque<>(); void push(int val) { int curMin = st.isEmpty() ? val : Math.min(val, st.peek()[1]); st.push(new int[]{val, curMin}); } void pop() { st.pop(); } int top() { return st.peek()[0]; } int getMin(){ return st.peek()[1]; } // O(1)! // Queue using Two Stacks (amortized O(1)) Deque<Integer> inSt = new ArrayDeque<>(); Deque<Integer> outSt = new ArrayDeque<>(); void enqueue(int x) { inSt.push(x); } int dequeue() { if (outSt.isEmpty()) // lazy pour while (!inSt.isEmpty()) outSt.push(inSt.pop()); return outSt.pop(); } // Flatten Nested List Iterator — use stack of iterators Deque<Iterator<NestedInteger>> stk = new ArrayDeque<>(); stk.push(nestedList.iterator()); // hasNext: peek until integer found, flatten lists lazily
Pattern: Augment each stack entry with auxiliary info (min, max). Two-stack queue: push to IN, pop from OUT (pour lazily). Never pour when OUT is non-empty.

Postorder iterative trick: do reverse-preorder (Root→Right→Left) then reverse the result. Or use two stacks.
Inorder + Preorder + Postorder iterative
// INORDER iterative (Left → Root → Right) Deque<TreeNode> st = new ArrayDeque<>(); TreeNode cur = root; while (cur!=null || !st.isEmpty()) { while (cur!=null) { st.push(cur); cur=cur.left; } cur = st.pop(); result.add(cur.val); // VISIT cur = cur.right; } // PREORDER iterative (Root → Left → Right) st.push(root); while (!st.isEmpty()) { TreeNode n = st.pop(); result.add(n.val); // VISIT if (n.right!=null) st.push(n.right); // right first! if (n.left !=null) st.push(n.left); } // POSTORDER iterative (Left → Right → Root) // = reverse of (Root → Right → Left) st.push(root); LinkedList<Integer> post = new LinkedList<>(); while (!st.isEmpty()) { TreeNode n = st.pop(); post.addFirst(n.val); // add to FRONT if (n.left !=null) st.push(n.left); if (n.right!=null) st.push(n.right); }
Pattern: Inorder = go left until null, pop & visit, go right. Preorder = push right then left. Postorder = reverse preorder (push left then right, addFirst to result).
