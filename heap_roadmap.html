HEAP / PRIORITY QUEUE

Top-K, Kth element, two-heap median
CORE IDEA: A heap gives O(log n) insert and O(1) peek of the min/max. Reach for it on "top K", "Kth largest", "merge K sorted", and streaming median problems.
1
Top-K / Kth Element
Time O(n log k)Space O(k)
Questions
1.Kth Largest Element in an Array
2.Top K Frequent Elements
3.K Closest Points to Origin
4.Task Scheduler
Keyword: "top K", "K largest/smallest", "K closest", "K most frequent". Keep a size-K heap of the opposite extreme.
Pseudocode
// Kth largest → min-heap of size k PriorityQueue<Integer> pq = new PriorityQueue<>(); // min-heap for (int num : nums) { pq.offer(num); if (pq.size() > k) pq.poll(); // drop smallest } return pq.peek(); // kth largest = heap top // Max-heap: new PriorityQueue<>(Collections.reverseOrder()) // Custom: (a,b)->a[0]-b[0] (min by first field)
Pattern: For Kth largest, keep a min-heap of size K — the root is your answer. This is O(n log k), better than sorting's O(n log n).
2
Two Heaps / Merge K
Time O(N log k)Space O(k)
Questions
1.Find Median from Data Stream
2.Merge K Sorted Lists
3.Smallest Range Covering K Lists
4.Sliding Window Median
Keyword: "median of stream", "merge K sorted", "balance two halves". Max-heap for lower half, min-heap for upper half.
Pseudocode
// Median: maxHeap (low half) + minHeap (high half) PriorityQueue<Integer> lo = new PriorityQueue<>(Collections.reverseOrder()); PriorityQueue<Integer> hi = new PriorityQueue<>(); void add(int num) { lo.offer(num); hi.offer(lo.poll()); // balance values if (hi.size() > lo.size()) // balance sizes lo.offer(hi.poll()); } double median() { return lo.size()>hi.size() ? lo.peek() : (lo.peek()+hi.peek())/2.0; }
Pattern: Keep heaps balanced (sizes differ by ≤1). Median is the top of the larger heap, or the average of both tops. Merge K: push list heads, pop-and-push next.
3
K-Way Merge & Sorted Streams
Time O(N log k)Space O(k)
Questions
1.Merge K Sorted Lists
2.Kth Smallest Element in a Sorted Matrix
3.Find K Pairs with Smallest Sums
4.Smallest Range Covering K Lists
5.Ugly Number II
Keyword: "k sorted lists", "sorted matrix", "smallest next candidate". Heap stores the current frontier from each source.
Pseudocode
// K-way merge PriorityQueue<Node> pq = new PriorityQueue<>((a,b)->a.val-b.val); for (ListNode head : lists) if (head != null) pq.offer(head); while (!pq.isEmpty()) { ListNode cur = pq.poll(); tail.next = cur; tail = tail.next; if (cur.next != null) pq.offer(cur.next); }
Pattern: Push one item per sorted source. Pop the best item, then push the next item from the same source.
4
Scheduling, Intervals & Delayed Removal
Time O(n log n)Space O(n)
Questions
1.Meeting Rooms II
2.Meeting Rooms III
3.Employee Free Time
4.Process Tasks Using Servers
5.Sliding Window Median
6.Maximize Score After K Operations
Keyword: "earliest ending", "available server", "median with deletes". Heaps model the next resource to free or the current best resource.
Pseudocode
// Meeting Rooms II Arrays.sort(intervals, (a,b)->a[0]-b[0]); PriorityQueue<Integer> ends = new PriorityQueue<>(); for (int[] mtg : intervals) { if (!ends.isEmpty() && ends.peek() <= mtg[0]) ends.poll(); ends.offer(mtg[1]); } return ends.size(); // Sliding median: two heaps + lazy delete map
Pattern: Min-heap of end times tells which resource frees first. For arbitrary deletes, mark invalid entries and clean heap tops lazily.
OperationTimeNote
offer / pollO(log n)peek is O(1)
Top-K with size-K heapO(n log k)beats full sort
Build heap from arrayO(n)heapify, not n inserts
