BIT MANIPULATION

XOR tricks, masking, set-bit counting
CORE IDEA: Operate directly on binary representation. XOR cancels pairs, AND masks bits, shifts multiply/divide by powers of 2. Often turns O(n) space into O(1).
1
XOR Tricks
Time O(n)Space O(1)
Questions
1.Single Number
2.Single Number II & III
3.Missing Number
4.Find the Difference
Keyword: "every element appears twice except one", "find missing/duplicate", "without extra space". XOR of a number with itself = 0.
Pseudocode
// Single Number — XOR cancels pairs int x = 0; for (int num : nums) x ^= num; return x; // duplicates cancel, lone number remains // Missing Number in [0..n] int missing = n; for (int i=0; i<n; i++) missing ^= i ^ nums[i]; return missing; // Key identities: // a^a = 0, a^0 = a, XOR is commutative
Pattern: XOR all elements — anything appearing an even number of times cancels to 0, leaving the odd one out.

Keyword: "count set bits", "is power of 2", "generate all subsets", "toggle/check bit". n & (n-1) clears the lowest set bit.
Pseudocode
// Count set bits — Brian Kernighan int count = 0; while (n != 0) { n &= (n - 1); // clears lowest set bit count++; } // Essential bit operations: // check bit i: (n >> i) & 1 // set bit i: n | (1 << i) // clear bit i: n & ~(1 << i) // toggle bit i: n ^ (1 << i) // is power of 2: n > 0 && (n & (n-1)) == 0 // Enumerate all subsets of n elements: for (int mask=0; mask<(1<<n); mask++) { /* ... */ }
Pattern: n & (n-1) removes the lowest set bit — counts bits in O(set bits). Bitmask loops 0..2ⁿ enumerate every subset.

Bitmask Enumeration & DP
Time O(2^n * n)Space O(2^n)
Questions
1.Subsets
2.Partition to K Equal Sum Subsets
3.Shortest Path Visiting All Nodes
4.Minimum XOR Sum of Two Arrays
5.Stickers to Spell Word
6.Maximum Product of Word Lengths
Keyword: "n <= 20", "visited subset", "choose unused item", "word character mask". A bitmask compresses a set into one integer.
Pseudocode
// DP over subsets int FULL = (1 << n) - 1; int[] dp = new int[1 << n]; Arrays.fill(dp, INF); dp[0] = 0; for (int mask=0; mask<=FULL; mask++) { for (int i=0; i<n; i++) { if (((mask >> i) & 1) == 1) continue; int next = mask | (1 << i); dp[next] = Math.min(dp[next], dp[mask] + cost(mask, i)); } }
Pattern: State is the set of chosen/visited items. Transitions add, remove, or test one bit.
4
Bitwise AND/OR Ranges & Binary Trie
Time O(n*32)Space O(n*32)
Questions
1.Bitwise AND of Numbers Range
2.Maximum XOR of Two Numbers in an Array
3.Maximum XOR With an Element From Array
4.Total Hamming Distance
5.Sum of Two Integers
Keyword: "max XOR", "AND of range", "without + operator", "pair bit contribution". Think bit-by-bit, usually 31..0.
Pseudocode
// Range AND: remove changing suffix bits int shift = 0; while (left < right) { left >>= 1; right >>= 1; shift++; } return left << shift; // Max XOR: for each bit, prefer opposite branch in trie for (int bit=31; bit>=0; bit--) { int want = 1 - ((num >> bit) & 1); // take want child if it exists, else same bit }
Pattern: Range AND keeps only the common binary prefix. XOR is maximized by choosing opposite bits greedily from high to low.
