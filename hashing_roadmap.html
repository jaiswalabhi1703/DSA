HASHING

Frequency maps, complement lookup, sets
CORE IDEA: Trade space for time. HashMap gives O(1) average lookup. Use it for frequency counting, seen-before checks, grouping, and complement lookups.
1
Frequency & Complement Lookup
Time O(n)Space O(n)
Questions
1.Two Sum
2.Group Anagrams
3.Top K Frequent Elements
4.Valid Anagram
5.First Unique Character
Keyword: "find pair", "count occurrences", "group by", "have I seen this before", "most/least frequent". Map key = signature, value = index/count/list.
Pseudocode
// Two Sum — complement lookup Map<Integer,Integer> seen = new HashMap<>(); for (int i=0; i<n; i++) { int need = target - nums[i]; if (seen.containsKey(need)) return new int[]{seen.get(need), i}; seen.put(nums[i], i); } // Group Anagrams — sorted string as key Map<String,List<String>> map = new HashMap<>(); for (String s : strs) { char[] c = s.toCharArray(); Arrays.sort(c); String key = new String(c); map.computeIfAbsent(key, k->new ArrayList<>()).add(s); }
Pattern: Build a map in one pass. Key design is everything: sorted string for anagrams, char-count array for fingerprints, prefix sum for subarray problems.
2
HashSet for O(1) Membership
Time O(n)Space O(n)
Questions
1.Longest Consecutive Sequence
2.Contains Duplicate
3.Intersection of Two Arrays
4.Happy Number
Keyword: "distinct", "consecutive", "is x present", "no duplicates". Set membership in O(1) replaces nested loops.
Pseudocode
// Longest Consecutive Sequence — O(n) Set<Integer> set = new HashSet<>(Arrays.asList(...)); int best = 0; for (int num : set) { if (!set.contains(num-1)) { // only start of a run int cur = num, len = 1; while (set.contains(cur+1)) { cur++; len++; } best = Math.max(best, len); } } // Each number visited at most twice → O(n)
Pattern: Only start counting from a sequence's beginning (when num-1 absent). This keeps total work O(n) despite the inner while.
