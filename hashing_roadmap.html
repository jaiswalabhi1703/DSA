HASHING ROADMAP

HashMap, HashSet, prefix sums, signatures, and cache-style design
CORE IDEA: Trade space for time. HashMap and HashSet give O(1) average lookup. Use hashing for frequency counting, seen-before checks, complement lookup, grouping by signature, prefix-sum history, and fast state lookup.
Hashing Pattern Recognition
  • "pair with target" or "needed value" -> complement map
  • "count occurrences" or "most frequent" -> frequency map
  • "distinct", "duplicate", "visited" -> HashSet
  • "subarray sum equals k" -> prefix sum + map
  • "same structure" or "group similar" -> signature key
  • "range/nearest key" -> TreeMap / TreeSet
Key Design Checklist
  • Number key: value, index, prefix sum, remainder
  • String key: sorted chars, char counts, normalized pattern
  • Composite key: row + col, pair, encoded state
  • Map value: count, first index, last index, list, heap, node
  • Collision-safe mindset: average O(1), worst O(n)

Keyword: "find pair", "count occurrences", "appears once", "most frequent", "needed value". Map key = item, value = count/index/list.
Pseudocode
// Two Sum - complement lookup Map<Integer,Integer> seen = new HashMap<>(); for (int i=0; i<n; i++) { int need = target - nums[i]; if (seen.containsKey(need)) return new int[]{seen.get(need), i}; seen.put(nums[i], i); } // Frequency count Map<Integer,Integer> freq = new HashMap<>(); for (int x : nums) freq.put(x, freq.getOrDefault(x, 0) + 1); // Top K: count first, then heap/bucket sort
Pattern: Ask what information you wish you already had. Store that in the map while scanning once.

Keyword: "distinct", "duplicate", "visited", "is x present", "cycle by repeated state". Set membership replaces nested loops.
Pseudocode
// Longest Consecutive Sequence Set<Integer> set = new HashSet<>(); for (int x : nums) set.add(x); int best = 0; for (int x : set) { if (!set.contains(x - 1)) { int cur = x, len = 1; while (set.contains(cur + 1)) { cur++; len++; } best = Math.max(best, len); } }
Pattern: Only start a consecutive run when x-1 is absent. Every number participates in at most one expansion.

Keyword: "subarray sum", "sum divisible", "equal 0 and 1", "count ranges". Store previous prefix sums or remainders.
Pseudocode
// Count subarrays with sum exactly k Map<Integer,Integer> count = new HashMap<>(); count.put(0, 1); int prefix = 0, ans = 0; for (int x : nums) { prefix += x; ans += count.getOrDefault(prefix - k, 0); count.put(prefix, count.getOrDefault(prefix, 0) + 1); } // Divisible by k: use remainder key int rem = ((prefix % k) + k) % k; // Longest length: store first index of prefix
Pattern: Current prefix minus old prefix equals target. For counting, store frequencies. For longest length, store first index only.

Keyword: "substring/window", "at most k distinct", "same frequency", "contains all chars". HashMap tracks window counts.
Pseudocode
// At most k distinct Map<Character,Integer> freq = new HashMap<>(); int left = 0, best = 0; for (int right=0; right<s.length(); right++) { char c = s.charAt(right); freq.put(c, freq.getOrDefault(c, 0) + 1); while (freq.size() > k) { char out = s.charAt(left++); freq.put(out, freq.get(out) - 1); if (freq.get(out) == 0) freq.remove(out); } best = Math.max(best, right - left + 1); }
Pattern: Expand right, update count, shrink left while invalid. For exact K, often use atMost(K) - atMost(K-1).

Keyword: "group same", "same pattern", "same contents", "isomorphic". Build a stable key representing the real identity.
Pseudocode
// Group Anagrams - char-count signature Map<String,List<String>> groups = new HashMap<>(); for (String s : strs) { int[] cnt = new int[26]; for (char c : s.toCharArray()) cnt[c - 'a']++; StringBuilder key = new StringBuilder(); for (int x : cnt) key.append('#').append(x); groups.computeIfAbsent(key.toString(), z -> new ArrayList<>()).add(s); } // Isomorphic: keep two maps for both directions
Pattern: The key must preserve exactly the equality you care about. Sorted string works; count-array key is often faster.

Pseudocode
// Contains Duplicate III - sliding ordered set TreeSet<Long> set = new TreeSet<>(); for (int i=0; i<nums.length; i++) { long x = nums[i]; Long near = set.ceiling(x - t); if (near != null && near <= x + t) return true; set.add(x); if (i >= k) set.remove((long) nums[i-k]); } return false; // TreeMap: floorKey(), ceilingKey(), higherKey(), lowerKey()
Pattern: HashMap is not enough when you need nearest key or sorted iteration. Switch to TreeMap/TreeSet and accept O(log n).

Keyword: "O(1) get/put/delete", "cache", "random access", "latest state". Combine map with list, array, heap, or deque.
Pseudocode
// LRU Cache idea Map<Integer,Node> map = new HashMap<>(); DoublyLinkedList list = new DoublyLinkedList(); int get(int key) { if (!map.containsKey(key)) return -1; Node node = map.get(key); list.moveToFront(node); return node.val; } void put(int key, int val) { if (map.containsKey(key)) { Node node = map.get(key); node.val = val; list.moveToFront(node); } else { Node node = new Node(key, val); map.put(key, node); list.addFront(node); if (map.size() > cap) evictTail(); } }
Pattern: The map finds the object in O(1). The companion structure handles ordering, frequency, random index, or eviction.
