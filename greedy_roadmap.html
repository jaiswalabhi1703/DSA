GREEDY

Interval scheduling, jumps, allocation
CORE IDEA: Make the locally optimal choice at each step and never reconsider. Works only when a "greedy choice property" holds — usually after sorting. Always sanity-check with a counterexample.
1
Interval Scheduling
Time O(n log n)Space O(1)
Questions
1.Non-overlapping Intervals
2.Merge Intervals
3.Meeting Rooms II
4.Minimum Number of Arrows to Burst Balloons
Keyword: "intervals", "non-overlapping", "merge", "minimum rooms/arrows". Sort by end time (for max count) or start time (for merging).
Pseudocode
// Max non-overlapping intervals → sort by END Arrays.sort(intervals, (a,b)->a[1]-b[1]); int end = Integer.MIN_VALUE, count = 0; for (int[] it : intervals) { if (it[0] >= end) { // no overlap → take it count++; end = it[1]; } } // removals = total - count // Merge → sort by START, extend or push Arrays.sort(intervals, (a,b)->a[0]-b[0]); // if cur.start <= last.end → merge, else add new
Pattern: Sort by end time, greedily pick the interval that finishes earliest — it leaves the most room. Sort by start time for merging.
2
Greedy Choice / Jump / Allocation
Time O(n)/O(n log n)Space O(1)
Questions
1.Jump Game I & II
2.Gas Station
3.Candy
4.Partition Labels
5.Assign Cookies
Keyword: "minimum jumps", "can you reach", "fewest/most...", "distribute fairly". Track a running max-reach or best-so-far.
Pseudocode
// Jump Game — can you reach the end? int maxReach = 0; for (int i=0; i<n; i++) { if (i > maxReach) return false; // stuck maxReach = Math.max(maxReach, i + nums[i]); } return true; // Jump Game II — fewest jumps (BFS-like greedy) int jumps=0, curEnd=0, farthest=0; for (int i=0; i<n-1; i++) { farthest = Math.max(farthest, i+nums[i]); if (i == curEnd) { jumps++; curEnd = farthest; } }
Pattern: Maintain the farthest reachable index. Jump only when forced (current window exhausted). Proof of correctness is the hard part — test edge cases.
StepTimeNote
Sort (usual first step)O(n log n)by start or end

With heap (Meeting Rooms II)O(n log n)min-heap of end times
