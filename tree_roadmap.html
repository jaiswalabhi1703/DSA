TREE DFS ROADMAP

All tree traversal patterns · 8 phases · Java pseudocode + complexity
Core Idea

DFS on a tree is all about how information flows in the recursion.
Either the parent gives state to children (Top-Down) or children return answers to the parent (Bottom-Up).
Choose the direction first, then write the recurrence.

Traversal Orders
InorderLeft → Root → Right (sorted for BST)
PreorderRoot → Left → Right (top-down)
PostorderLeft → Right → Root (bottom-up)
When to use which?
  • Preorder → parent passes info to children
  • Postorder → children compute, parent combines
  • Inorder → sorted order / BST validation
  • Backtracking DFS → explore all root-to-leaf paths
1
Basic Traversal & Simple DFS
Time O(n)Space O(h)
Questions
1.InorderInorder / Preorder / Postorder Traversal
2.PostorderMaximum Depth of Binary Tree
3.PreorderSame Tree
4.PreorderSymmetric Tree (mirror)
5.PreorderInvert Binary Tree
Keyword: "traverse the tree", "depth/height", "compare two trees", "mirror/invert". The plain recursion skeleton covers all of these.
Pseudocode
// The universal DFS shape void dfs(TreeNode root) { if (root == null) return; // base case // preorder: visit root here dfs(root.left); // inorder: visit root here dfs(root.right); // postorder: visit root here } // Maximum Depth (postorder) int depth(TreeNode root) { if (root==null) return 0; return 1 + Math.max(depth(root.left), depth(root.right)); }
Pattern: Where you place the "visit" line determines the order. Every node is visited once → O(n) time, O(h) recursion stack.
2
Bottom-Up DFS (Subtree Aggregation)
Time O(n)Space O(h)
Questions
1.PostorderBalanced Binary Tree
2.PostorderDiameter of Binary Tree
3.PostorderBinary Tree Tilt
4.PostorderLongest Univalue Path
Keyword: "diameter", "balanced", "longest path", "sum/aggregate of subtree". Children compute first, parent combines their answers.
Pseudocode
// Diameter — return height, update global diameter int diameter = 0; int height(TreeNode root) { if (root==null) return 0; int l = height(root.left); int r = height(root.right); diameter = Math.max(diameter, l + r); // path thru node return 1 + Math.max(l, r); // height up } // Balanced: return -1 sentinel if any subtree unbalanced
Pattern: Children compute first → parent combines. Return one value upward; track cross-subtree answers in a global variable.
3
Top-Down DFS (State Passing)
Time O(n)Space O(h)
Questions
1.PreorderPath Sum I & II
2.PreorderBinary Tree Paths
3.PreorderSum Root to Leaf Numbers
4.PreorderCount Good Nodes in Binary Tree
5.PreorderPseudo-Palindromic Paths
Keyword: "root-to-leaf", "path sum equals target", "carry value down the path", "good nodes". Parent passes accumulated state to children.
Pseudocode
// Path Sum — pass remaining target downward boolean hasPathSum(TreeNode root, int target) { if (root==null) return false; target -= root.val; // state down if (root.left==null && root.right==null) return target == 0; // leaf check return hasPathSum(root.left, target) || hasPathSum(root.right, target); } // Good Nodes: pass max-so-far down the path
Pattern: Parent passes state (running sum, path, max, depth, freq) down to children as a parameter. Decision often made at the leaf.
4
BST DFS Patterns
Time O(h)→O(n)Space O(h)
Questions
1.RangeValidate BST
2.InorderKth Smallest in BST
3.BST PropLowest Common Ancestor of BST
4.DivideConvert Sorted Array to BST
5.InorderRecover BST
Keyword: "BST", "validate", "kth smallest", "sorted order". Use the BST property: left < node < right. Inorder yields sorted values.
Pseudocode
// Validate BST — pass valid (low, high) range down boolean valid(TreeNode node, long low, long high) { if (node==null) return true; if (node.val<=low || node.val>=high) return false; return valid(node.left, low, node.val) && valid(node.right, node.val, high); } // LCA of BST — use ordering to pick a direction while (node!=null) { if (p.val<node.val && q.val<node.val) node=node.left; else if (p.val>node.val && q.val>node.val) node=node.right; else return node; // split point = LCA }
Pattern: Exploit ordering. Validate with a shrinking (low, high) range. Inorder traversal of a BST is sorted — use it for Kth smallest and Recover BST.
5
Lowest Common Ancestor (LCA)
Time O(n)Space O(h)
Questions
1.PostorderLCA of Binary Tree
2.BST PropLCA of BST
3.PostorderLCA with Parent Pointers
4.PostorderSmallest Subtree with all Deepest Nodes
Keyword: "lowest common ancestor", "deepest node covering both", "meeting point of two nodes". Return whether each target exists in the subtree.
Pseudocode
// LCA of Binary Tree — postorder, bubble up findings TreeNode lca(TreeNode root, TreeNode p, TreeNode q) { if (root==null || root==p || root==q) return root; TreeNode l = lca(root.left, p, q); TreeNode r = lca(root.right, p, q); if (l!=null && r!=null) return root; // split → LCA return l!=null ? l : r; // pass found up }
Pattern: Return whether a target exists in each subtree. The node where left and right both return non-null is the LCA.
6
Tree DP (Advanced — Postorder)
Time O(n)Space O(h)
Questions
1.Tree DPHouse Robber III
2.Tree DPBinary Tree Maximum Path Sum
3.Tree DPBinary Tree Cameras
4.Tree DPDistribute Coins in Binary Tree
Keyword: "max path sum", "rob the tree", "minimum cameras", "balance subtrees". DFS returns multiple states/values to the parent.
Pseudocode
// House Robber III — return {rob, skip} int[] dfs(TreeNode node) { if (node==null) return new int[]{0,0}; int[] l = dfs(node.left); int[] r = dfs(node.right); int rob = node.val + l[1] + r[1]; // take node int skip = Math.max(l[0],l[1]) + Math.max(r[0],r[1]); // skip node return new int[]{rob, skip}; } // Max Path Sum: return one-side gain, update global best
Pattern: DFS returns multiple states (rob/skip, covered/notCovered). Parent picks the best combination. Update a global for the overall answer.

Serialization & Construction
Time O(n)Space O(n)
Questions
1.PreorderSerialize & Deserialize Binary Tree
2.Pre+InConstruct BT from Preorder & Inorder
3.Post+InConstruct BT from Postorder & Inorder
4.PreorderFlatten Binary Tree to Linked List
Keyword: "serialize/deserialize", "construct tree from traversals", "encode tree as string". Preorder + null markers, or split using inorder index.
Pseudocode
// Serialize (preorder with null markers) void ser(TreeNode n, StringBuilder sb) { if (n==null) { sb.append("#,"); return; } sb.append(n.val).append(","); ser(n.left, sb); ser(n.right, sb); } // Construct from preorder + inorder // 1. preorder[preIdx] = root // 2. find root in inorder → splits L | R subtrees // 3. recurse left then right (use a HashMap for index) Map<Integer,Integer> idx; // value → inorder index, O(1) lookup
Pattern: Reconstruct or encode. Preorder gives the root order; inorder gives the left/right split. A HashMap of inorder indices makes lookup O(1).
8
Tree as a Graph (edges given)
Time O(n)Space O(n)
Questions
1.DFSCount Subtrees With Same Label
2.DFSMinimum Time to Collect All Apples
3.DFSSum of Distances in Tree (rerooting)
4.DFSTree Diameter (from edge list)
Keyword: tree given as an edge list not a TreeNode, "n nodes and n-1 edges", "rooted at 0". Build adjacency list, DFS with a parent guard.
Pseudocode
// Tree given as edges → build adjacency list List<Integer>[] adj = new List[n]; for (int[] e : edges) { // undirected adj[e[0]].add(e[1]); adj[e[1]].add(e[0]); } void dfs(int node, int parent) { process(node); for (int nei : adj[node]) if (nei != parent) // avoid going back up dfs(nei, node); }
Pattern: When a tree is given as edges, build an adjacency list and DFS with a parent argument so you never walk back up. Re-rooting handles "for every node" queries in O(n).
GOLDEN RULE: First decide what information each node needs.
Then choose the direction of flow — Top-Down (preorder) or Bottom-Up (postorder).
Master DFS on trees → master 50% of DSA.
