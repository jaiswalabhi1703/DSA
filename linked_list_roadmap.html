LINKED LIST

Reversal, fast/slow, dummy head
CORE IDEA: Master pointer manipulation. Use a dummy head to simplify edge cases, fast/slow for the middle & cycles, and careful next-pointer rewiring for reversal.
1
Reversal Patterns
Time O(n)Space O(1)
Questions
1.Reverse Linked List
2.Reverse Linked List II (range)
3.Reverse Nodes in k-Group
4.Swap Nodes in Pairs
5.Palindrome Linked List
Keyword: "reverse the list", "reverse in groups of k", "swap pairs". Three-pointer dance: prev, cur, next.
Pseudocode
// Iterative reversal — prev/cur/next ListNode prev = null, cur = head; while (cur != null) { ListNode next = cur.next; // save cur.next = prev; // reverse pointer prev = cur; // advance prev cur = next; // advance cur } return prev; // new head // k-Group: count k nodes ahead, reverse, recurse on rest
Pattern: Always save cur.next before rewiring. prev becomes the new head after the loop.
2
Fast & Slow + Dummy Head
Time O(n)Space O(1)
Questions
1.Middle of the Linked List
2.Linked List Cycle I & II
3.Remove Nth Node From End
4.Merge Two Sorted Lists
5.Remove Duplicates from Sorted List
Keyword: "find the middle", "detect cycle", "nth from end", "merge lists", "remove node". Use dummy head when the head itself may change.
Pseudocode
// Floyd's cycle detection ListNode slow=head, fast=head; while (fast!=null && fast.next!=null) { slow = slow.next; // 1 step fast = fast.next.next; // 2 steps if (slow==fast) break; // cycle found } // cycle start: reset one ptr to head, move both 1 step // Dummy head removes head edge cases ListNode dummy = new ListNode(0); dummy.next = head; // ... operate, return dummy.next
Pattern: Fast moves 2×, slow 1×. When fast hits the end, slow is at the middle. For Nth-from-end: advance fast N nodes first, then move both.
3
Merge, Sort & Divide-Conquer Lists
Time O(n log n)Space O(log n)
Questions
1.Merge Two Sorted Lists
2.Merge K Sorted Lists
3.Sort List
4.Insertion Sort List
5.Partition List
6.Convert Sorted List to BST
Keyword: "sorted lists", "merge k", "sort linked list in O(n log n)". Use dummy heads, split with slow/fast, merge sorted halves.
Pseudocode
// Merge sort on linked list ListNode sortList(ListNode head) { if (head == null || head.next == null) return head; ListNode mid = splitMiddle(head); ListNode right = mid.next; mid.next = null; ListNode a = sortList(head); ListNode b = sortList(right); return merge(a, b); } // Merge K: priority queue or divide and conquer pairwise merge
Pattern: Linked-list sorting is merge sort, not quicksort. Splitting and merging only rewires next pointers.
4
Reorder, Copy & Pointer Surgery
Time O(n)Space O(1)/O(n)
Questions
1.Reorder List
2.Rotate List
3.Odd Even Linked List
4.Copy List with Random Pointer
5.Flatten a Multilevel Doubly Linked List
6.Add Two Numbers I & II
Keyword: "reorder in-place", "random pointer", "rotate", "flatten". Draw nodes; save next pointers before rewiring.
Pseudocode
// Reorder List: L0, Ln, L1, Ln-1... ListNode mid = middle(head); ListNode second = reverse(mid.next); mid.next = null; ListNode first = head; while (second != null) { ListNode fNext = first.next; ListNode sNext = second.next; first.next = second; second.next = fNext; first = fNext; second = sNext; } // Copy random pointer: HashMap old->new, or weave cloned nodes
Pattern: Many hard list problems are combinations of middle finding, reversing the second half, and merging two chains.
