DYNAMIC PROGRAMMING ROADMAP (PHASE-WISE)

Click a phase to focus · 8 phases with Java-style pseudocode
CORE IDEA

DP = break problem into overlapping subproblems + cache results.
Either Top-Down (Memoization) → recursion + HashMap/array cache
or Bottom-Up (Tabulation) → fill dp[] table iteratively.
Golden question: "What is dp[i] / dp[i][j] representing?"

DP Pattern Recognition
  • 1D DP → single array, index = state
  • 2D DP → two sequences / grid → dp[i][j]
  • Interval DP → dp[l][r] on subarray ranges
  • String DP → edit distance, LCS, palindrome
  • Tree DP → postorder, aggregate from children
  • Bitmask DP → subset states, dp[mask]
  • Greedy+DP → sort then DP (LIS, activity)
When to use DP?
  • Overlapping subproblems exist
  • Optimal substructure holds
  • "Count ways" or "min/max" asked
  • Choices at each step affect future
  • Brute force is exponential O(2ⁿ)
Top-Down: easier to write, natural recursion
Bottom-Up: faster, no stack overflow

PHASE 1 · 1D DP — FOUNDATION
Linear DP · O(n)
Questions
1.1DClimbing Stairs
2.1DHouse Robber
3.1DHouse Robber II (circular)
4.1DMin Cost Climbing Stairs
5.1DFibonacci Number
6.Greedy+DPJump Game I & II
7.1DDecode Ways

dp[i] = answer for first i elements.
Transition looks back 1-2 steps.
Often space-optimizable to O(1) using two variables.
House Robber template
// dp[i] = max money robbing houses 0..i // dp[i] = max(dp[i-1], dp[i-2] + nums[i]) int rob(int[] nums) { int n = nums.length; if (n==1) return nums[0]; int prev2=nums[0], prev1=Math.max(nums[0],nums[1]); for (int i=2; i<n; i++) { int cur = Math.max(prev1, prev2+nums[i]); prev2 = prev1; prev1 = cur; } return prev1; } // Space: O(1) — rolling variables instead of dp[] // Climbing Stairs (same pattern): // dp[i] = dp[i-1] + dp[i-2] // Decode Ways: // dp[i] += dp[i-1] if valid 1-digit // dp[i] += dp[i-2] if valid 2-digit
Pattern: Define dp[i], write recurrence, handle base cases, optimize space with rolling vars.

PHASE 2 · KNAPSACK PATTERNS
0/1 · Unbounded · Subset Sum
Questions
1.0/1 KSPartition Equal Subset Sum
2.0/1 KSTarget Sum (assign +/-)
3.0/1 KSLast Stone Weight II
4.UnboundedCoin Change (min coins)
5.UnboundedCoin Change II (count ways)
6.UnboundedInteger Break
7.0/1 KSOnes and Zeroes (2D knapsack)

0/1: each item used once → iterate j backwards.
Unbounded: items reusable → iterate j forwards.
This single direction flip is the only difference.
0/1 KS + Unbounded templates
// 0/1 Knapsack — each item used once // dp[j] = can we reach sum j? boolean[] dp = new boolean[target+1]; dp[0] = true; for (int num : nums) for (int j=target; j>=num; j--) // ← BACKWARDS dp[j] |= dp[j-num]; // Unbounded Knapsack — item reusable // Coin Change: dp[j] = min coins to make j int[] dp2 = new int[amount+1]; Arrays.fill(dp2, amount+1); dp2[0] = 0; for (int coin : coins) for (int j=coin; j<=amount; j++) // → FORWARDS dp2[j] = Math.min(dp2[j], dp2[j-coin]+1); // Coin Change II (count ways): // dp[j] += dp[j-coin] (forwards)
Key: 0/1 KS = backwards loop. Unbounded = forwards loop. The loop direction IS the distinction.

PHASE 3 · 2D DP — GRID & SEQUENCES
dp[i][j] · Grid paths · LCS · LIS
Questions

LIS in O(n log n): patience sorting with binary search.
LCS diagonal rule: if match → dp[i-1][j-1]+1 else max(left, up).
LCS + LIS templates
// LCS — dp[i][j] = LCS of s1[0..i-1], s2[0..j-1] int[][] dp = new int[m+1][n+1]; for (int i=1; i<=m; i++) for (int j=1; j<=n; j++) if (s1.charAt(i-1)==s2.charAt(j-1)) dp[i][j] = dp[i-1][j-1]+1; // match else dp[i][j] = Math.max(dp[i-1][j], dp[i][j-1]); // LIS — O(n log n) patience sort List<Integer> tails = new ArrayList<>(); for (int num : nums) { int pos = Collections.binarySearch(tails, num); if (pos < 0) pos = -(pos+1); if (pos==tails.size()) tails.add(num); else tails.set(pos, num); } // return tails.size() = LIS length
Pattern: 2D dp[i][j] where i,j index into two sequences or two dimensions. Fill row by row. LIS: binary search on maintained tails array.

Edit Distance + Word Break
// Edit Distance — dp[i][j] = min ops to convert // word1[0..i-1] → word2[0..j-1] for (int i=0; i<=m; i++) dp[i][0]=i; for (int j=0; j<=n; j++) dp[0][j]=j; for (int i=1; i<=m; i++) for (int j=1; j<=n; j++) if (w1.charAt(i-1)==w2.charAt(j-1)) dp[i][j] = dp[i-1][j-1]; // no op else dp[i][j] = 1+Math.min(dp[i-1][j-1], // replace Math.min(dp[i-1][j], // delete dp[i][j-1])); // insert // Word Break — dp[i] = can s[0..i-1] be segmented? boolean[] dp2 = new boolean[n+1]; dp2[0]=true; for (int i=1; i<=n; i++) for (String w : dict) if (i>=w.length() && dp2[i-w.length()] && s.substring(i-w.length(),i).equals(w)) dp2[i] = true;

PHASE 5 · INTERVAL DP
dp[l][r] · Fill by length
Questions
1.IntervalBurst Balloons
2.IntervalMatrix Chain Multiplication
3.IntervalMinimum Cost to Cut a Stick
4.IntervalStrange Printer
5.IntervalLongest Palindromic Subsequence

Burst Balloons trick: think of k as the last balloon to burst in range [l,r], not the first. This avoids dependency issues.
Interval DP template (Burst Balloons)
// dp[l][r] = max coins bursting all in (l,r) // k = last balloon to burst in range int[] nums = // pad with 1 at both ends int[][] dp = new int[n][n]; for (int len=1; len<=n-2; len++) { // fill by length for (int l=1; l<=n-2-len+1; l++) { int r = l+len-1; for (int k=l; k<=r; k++) { // try each last dp[l][r] = Math.max(dp[l][r], dp[l][k-1] // left part + nums[l-1]*nums[k]*nums[r+1] // burst k last + dp[k+1][r]); // right part } } } // Key: outer loop is LENGTH, not l or r!
Pattern: Outer loop = subarray length (1 → n). Inner loops: l from left, r = l+len-1, k iterates split points inside [l,r].

PHASE 6 · TREE DP
Postorder · aggregate from children
Questions
1.TreeHouse Robber III
2.TreeDiameter of Binary Tree
3.TreeBinary Tree Max Path Sum
4.TreeBinary Tree Cameras
5.TreeDistribute Coins in Binary Tree
6.TreeLongest Univalue Path

DFS returns value(s) for current subtree.
Parent combines children's answers.
Global answer updated inside DFS (not returned).
House Robber III + Max Path Sum
// House Robber III: dp returns int[]{rob, skip} int[] dfs(TreeNode node) { if (node==null) return new int[]{0,0}; int[] l = dfs(node.left); int[] r = dfs(node.right); int rob = node.val + l[1] + r[1]; // take node int skip = Math.max(l[0],l[1]) // skip node + Math.max(r[0],r[1]); return new int[]{rob, skip}; } // Max Path Sum: return single-path gain upward int maxSum = Integer.MIN_VALUE; int dfs2(TreeNode node) { if (node==null) return 0; int l = Math.max(0, dfs2(node.left)); // ignore negatives int r = Math.max(0, dfs2(node.right)); maxSum = Math.max(maxSum, node.val+l+r); // update global return node.val + Math.max(l, r); // return one side }
Pattern: DFS returns info for parent's use. Update a global variable for cross-subtree answers. Return only one path upward.

PHASE 7 · BITMASK DP
dp[mask] · Subset enumeration · n ≤ 20
Questions
1.BitmaskTravelling Salesman Problem
2.BitmaskShortest Path Visiting All Nodes
3.BitmaskPartition to K Equal Sum Subsets
4.BitmaskMinimum XOR Sum of Two Arrays
5.BitmaskStickers to Spell Word

Mask tricks:
Check bit i: (mask >> i) & 1
Set bit i: mask | (1 << i)
Unset bit i: mask & ~(1 << i)
Iterate subsets: for(sub=mask; sub>0; sub=(sub-1)&mask)
TSP / Bitmask DP template
// dp[mask][i] = min cost to visit nodes in mask // ending at node i int FULL = (1<<n)-1; int[][] dp = new int[1<<n][n]; for (int[] row : dp) Arrays.fill(row, INF); dp[1][0] = 0; // start at node 0, mask=0001 for (int mask=1; mask<=FULL; mask++) { for (int u=0; u<n; u++) { if (dp[mask][u]==INF) continue; if (((mask>>u)&1)==0) continue; // u not in mask for (int v=0; v<n; v++) { if ((mask>>v&1)==1) continue; // v already visited int nmask = mask|(1<<v); dp[nmask][v] = Math.min(dp[nmask][v], dp[mask][u]+dist[u][v]); } } } // answer: min(dp[FULL][i] + dist[i][0]) for all i
Pattern: State = (visited set as bitmask, current position). Transitions add one new node. Works when n ≤ 20. Time: O(2ⁿ · n²).

PHASE 8 · STOCK / STATE MACHINE DP
Finite states · transitions
Questions
1.StateBest Time to Buy/Sell Stock I
2.StateStock II (unlimited transactions)
3.StateStock III (at most 2 transactions)
4.StateStock IV (at most k transactions)
5.StateStock with Cooldown
6.StateStock with Transaction Fee

States: hold (own stock) / free (no stock, can buy).
Cooldown adds a 3rd state: cool (sold yesterday).
k-transactions: add k as a DP dimension.
State machine template (with cooldown)
// 3 states: hold, free, cool // hold = currently holding stock // free = not holding, can buy // cool = just sold, must wait 1 day int hold=Integer.MIN_VALUE, free=0, cool=0; for (int price : prices) { int prevHold=hold, prevFree=free, prevCool=cool; hold = Math.max(prevHold, // keep holding prevFree-price);// buy today free = Math.max(prevFree, // stay free prevCool); // cooldown expired cool = prevHold + price; // sell today } return Math.max(free, cool); // k-transactions: dp[k][0/1] where 0=not holding, 1=holding // dp[j][1] = max(dp[j][1], dp[j-1][0] - price) // buy // dp[j][0] = max(dp[j][0], dp[j][1] + price) // sell
Pattern: Model each DP state as a node, transitions as edges. Update all states simultaneously using previous day's values (don't use today's updated values mid-loop).
GOLDEN RULE: First define what dp[i] or dp[i][j] represents in plain English.
Then write the recurrence. Then handle base cases. Then optimize space.
